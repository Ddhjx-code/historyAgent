# 口播稿解析规则

## 概述

本文档定义了如何从任意口播稿中提取结构化信息，为后续的镜头生成和 Prompt 生成提供基础数据。

## 核心原则

1. **通用性**：不依赖特定格式或模板，适用于各类口播稿
2. **动态提取**：基于文本内容和上下文进行智能分析
3. **保留原始**：不修改口播稿原文，完整保留旁白内容
4. **结构识别**：识别段落、章节、标题等结构信息

## 解析流程

### 1. 段落结构识别

**目标**：识别口播稿的自然段落和结构标记

**识别规则**：
- **空行分隔**：连续的文本块为一个段落
- **标题标记**：以 `【` 开头的内容为标题（如 `【开场白】`、`【第一部分】`）
- **编号标记**：以数字开头的段落（如 `1.`、`2.`）

**输出结构**：
```json
{
  "sections": [
    {
      "type": "opening",
      "title": "【开场白】",
      "content": "大家好！欢迎收听...",
      "line_start": 1,
      "line_end": 5
    },
    {
      "type": "chapter",
      "title": "【第一部分：古代历史的见证】",
      "content": "亲爱的听众...",
      "line_start": 6,
      "line_end": 20
    }
  ]
}
```

---

### 2. 时间信息提取

**目标**：提取所有与时间相关的信息

**匹配模式**：

| 模式 | 示例 | 说明 |
|------|------|------|
| `公元(\d+)年` | 公元138年 | 提取年份 |
| `(\d+)年` | 1836年 | 提取年份 |
| `(\w+)朝` | 唐朝、明朝 | 提取朝代 |
| `(\w+)时代` | 工业时代、现代 | 提取时代 |
| `(\w+)世纪` | 19世纪、21世纪 | 提取世纪 |
| `(\d+)月(\d+)日` | 2月25日 | 提取日期 |

**提取规则**：
1. 使用正则表达式匹配上述模式
2. 保留完整的时间描述（如"公元138年"而非"138"）
3. 记录时间信息在文本中的位置

**输出结构**：
```json
{
  "time_info": [
    {
      "text": "公元138年",
      "value": "138",
      "type": "year_ad",
      "position": 15
    },
    {
      "text": "唐代",
      "value": "唐朝",
      "type": "dynasty",
      "position": 45
    }
  ]
}
```

---

### 3. 人物信息提取

**目标**：提取人物姓名、身份、动作描述

**匹配模式**：

| 模式 | 示例 | 说明 |
|------|------|------|
| `(\w+)皇帝` | 哈德良皇帝、唐太宗 | 皇帝 |
| `(\w+)高僧` | 玄奘高僧 | 僧人 |
| `(\w+)发明家` | 塞缪尔·柯尔特 | 发明家 |
| `(\w+)国王` | 埃里克十四世国王 | 国王 |
| `(\w+)苏丹` | 艾哈迈德二世苏丹 | 苏丹 |
| `(\w+)喇嘛` | 仓央嘉措喇嘛 | 喇嘛 |
| `(\w+)发言人` | 毛宁发言人 | 发言人 |

**提取规则**：
1. 使用正则表达式匹配人物模式
2. 提取人物姓名和身份
3. 从上下文中提取人物的动作描述

**输出结构**：
```json
{
  "characters": [
    {
      "name": "哈德良",
      "identity": "皇帝",
      "full_title": "哈德良皇帝",
      "actions": ["收养并指定安敦宁·毕尤为帝国的继承人"],
      "position": 20
    },
    {
      "name": "玄奘",
      "identity": "高僧",
      "full_title": "玄奘高僧",
      "actions": ["携带657部梵文佛经回到长安", "翻译佛经"],
      "position": 50
    }
  ]
}
```

---

### 4. 地点信息提取

**目标**：提取地名、场景类型、环境描述

**匹配模式**：

| 模式 | 示例 | 说明 |
|------|------|------|
| `(\w+)帝国` | 罗马帝国、波斯帝国 | 帝国 |
| `(\w+)王朝` | 萨珊王朝 | 王朝 |
| `(\w+)首都` | 长都、长安 | 首都 |
| `(\w+)王国` | 瑞典王国 | 王国 |
| `(\w+)地区` | 高加索地区 | 地区 |
| `(\w+)苏维埃` | 格鲁吉亚苏维埃 | 苏维埃政权 |

**提取规则**：
1. 使用正则表达式匹配地点模式
2. 提取地点名称和类型
3. 从上下文中提取环境描述

**输出结构**：
```json
{
  "locations": [
    {
      "name": "罗马帝国",
      "type": "empire",
      "description": "古罗马帝国的权力中心",
      "position": 18
    },
    {
      "name": "长安",
      "type": "capital",
      "description": "唐帝国的首都，也就是今天的西安",
      "position": 55
    }
  ]
}
```

---

### 5. 事件信息提取

**目标**：提取核心事件和事件描述

**识别规则**：
1. 查找动词密集的句子（如"发生了"、"建立了"、"发明了"）
2. 识别事件类型（权力交接、发明创造、建国独立等）
3. 提取事件的完整描述

**输出结构**：
```json
{
  "events": [
    {
      "type": "power_transition",
      "description": "哈德良收养并指定安敦宁·毕尤为帝国的继承人",
      "time": "公元138年",
      "location": "罗马帝国",
      "position": 22
    },
    {
      "type": "invention",
      "description": "塞缪尔·柯尔特改进了左轮手枪的设计",
      "time": "1836年",
      "location": "美国",
      "position": 80
    }
  ]
}
```

## 注意事项

1. **上下文依赖**：人物和地点的识别可能依赖上下文，需要结合前后文理解
2. **歧义处理**：对于模糊的描述，使用最合理的解释，或标记为需要人工确认
3. **信息完整性**：如果某些信息缺失（如没有明确的时间），使用合理的默认值或标记为 null
4. **多语言支持**：当前规则基于中文口播稿，如需支持其他语言，需要调整匹配模式
