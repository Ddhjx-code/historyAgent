# 口播稿解析规则

本文档定义大模型解析口播稿时应遵循的规则。

## 解析目标

从口播稿中提取：
1. **结构元素**：开场白、章节、事件、结尾
2. **事件元数据**：时间、人物、地点、时代
3. **情绪关键词**：识别文本中的情绪
4. **场景描述**：环境、氛围、光线

## 结构识别规则

### 标记识别

| 标记 | 类型 | 对应输出 |
|------|------|---------|
| `【开场白】`、`【开场】` | opening | 开场章节 |
| `【第X部分】` | transition | 章节转场 |
| `【第X个事件】`、`【事件X】` | event | 历史事件 |
| `【结尾】`、`【结语】` | ending | 结尾章节 |

### 内容累积规则

- 开场白：累积到章节类型为`opening`的section
- 章节：创建独立的transition section
- 事件：创建event对象，累积内容到event.content
- 结尾：累积到章节类型为`ending`的section

## 元数据提取规则

### 时间识别

**模式**：
- `1947年2月27日` → `date: 1947-02-27`
- `公元138年` → `year: 138`
- `19世纪` → `century: 19`

**正则表达式**：
```regex
(\d{4})年(\d{1,2})月(\d{1,2})日
公元(\d+)年
(\d+)世纪
```

### 人物识别

**模式**：
- `哈德良皇帝` → 角色 + 职位
- `林江迈，40岁寡妇` → 姓名 + 年龄 + 身份

**关键词**：
- 皇帝、国王、女王、高僧、寡妇、将军、大臣

### 地点识别

**模式**：
- `台北市太平町` → 地点 + 区域
- `罗马宫廷` → 地点 + 建筑
- `长安城门` → 地点 + 建筑

**行政区划**：
- 市、城、国、岛、省、州、县、镇、村
- 宫、殿、寺、庙、堂、馆、楼、阁

### 时代识别

**映射表**：

| 关键词 | 时代标识 |
|--------|---------|
| 台湾民国、1947 | `republic_of_china_1947` |
| 罗马、哈德良 | `ancient_rome` |
| 唐代、唐朝、玄奘 | `tang_dynasty` |
| 智利、现代 | `modern_chile` |
| 工业时代、19世纪 | `industrial_age` |
| 中世纪、骑士 | `medieval_europe` |

## 情绪识别规则

### 情绪类型映射

| 情绪类型 | 关键词 | 适用场景 |
|---------|--------|---------|
| SOLEMN | 庄严、肃穆、沉重、悲痛、威严 | 庄重历史、重大事件 |
| TENSE | 紧张、冲突、暴力、愤怒、压迫 | 冲突、对抗、危机 |
| HOPEFUL | 希望、振奋、成功、胜利、光明 | 希望、成功、转折 |
| SOMBER | 忧郁、悲凉、压抑、悲伤、苦难 | 悲伤、失败、压抑 |
| DRAMATIC | 戏剧性、转折、高潮、冲突、爆发 | 戏剧性高潮 |
| NEUTRAL | 平静、普通、正常、日常 | 中性叙述 |
| ANGRY | 愤怒、仇恨、仇视、暴怒 | 愤怒、仇恨 |
| PEACEFUL | 宁静、祥和、和平、安宁 | 和平、安宁 |

### 识别方法

1. 扫描文本中的情绪关键词
2. 统计每种情绪的出现次数
3. 返回出现次数最多的情绪类型
4. 如果没有情绪关键词，返回`NEUTRAL`

## 观众收获分析

### 观众收获类型

| 类型 | 关键词 | 说明 |
|------|--------|------|
| 信息 | 是、在、发生、叫做、名为、来自 | 获得事实信息 |
| 情绪 | 愤怒、悲伤、震惊、痛苦、绝望、希望 | 获得情绪体验 |
| 悬念 | 但、然而、突然、不料、竟然 | 产生悬念 |

### 分析规则

1. 检查文本中的关键词
2. 确定观众收获类型（可多个）
3. 用"、连接多个类型
4. 默认为"信息"

## 节拍提取规则

### 节拍定义

每个节拍是一个**最小叙事单元**，必须回答：
- **发生了什么**：具体的动作、事件、变化
- **观众得到什么**：信息（背景、事实）、情绪（愤怒、悲伤、希望）、悬念（未解之谜）

### 拆分规则

1. **按句子拆分**：以句号、感叹号、问号、分号为分隔符
2. **过滤短句**：忽略长度<10字符的句子
3. **分析观众收获**：为每个节拍确定观众收获
4. **保持完整性**：节拍应能独立理解

### 示例

**原文**：
```
林江迈在街边摆起了烟纸摊。查缉员出现在她面前，用枪托殴打她。
```

**节拍拆解**：
| 节拍 | 描述 | 观众收获 |
|------|------|---------|
| beat_001 | 林江迈在街边摆起了烟纸摊 | 信息：看到贫困寡妇为生存而挣扎 |
| beat_002 | 查缉员出现在她面前，用枪托殴打她 | 情绪：感受到官方暴力的残酷 |

## 输出格式

### ParsedScript结构

```yaml
sections:
  - type: "opening"
    content: "亲爱的听众朋友们，欢迎收听..."
    title: "开场白"
    emotion: "neutral"
events:
  - id: "event_001"
    title: "1947年2月27日 - 台湾'二二八事件'"
    content: "事件内容..."
    metadata:
      date: "1947-02-27"
      location: "台北市太平町"
      era: "republic_of_china_1947"
    beats: []
global_emotion: "somber"
metadata:
  total_sections: 3
  total_events: 1
```

## 注意事项

1. **保留原始内容**：不要修改或精简口播稿内容
2. **准确识别时代**：根据关键词准确判断时代
3. **情绪一致性**：同一事件的情绪应保持相对一致
4. **结构清晰**：确保输出的ParsedScript结构清晰可读
