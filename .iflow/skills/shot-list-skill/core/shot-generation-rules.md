# 镜头生成规则

## 概述

本文档定义了如何根据口播稿解析结果生成合理的镜头序列，包括镜头类型判断、时长分配、时间码计算等。

## 核心原则

1. **流畅性**：镜头序列应流畅自然，符合叙事节奏
2. **时长控制**：每个镜头时长 ≤ 15 秒（Seedance 硬性限制）
3. **类型合理**：根据内容选择合适的镜头类型
4. **时间连续**：时间码连续无断裂

---

## 镜头类型

### 类型定义

| 镜头类型 | 视频工具 | 用途 | 时长范围 |
|---------|---------|------|---------|
| title_card | ffmpeg | 节目标题、章节标题、结尾 | 6-10秒 |
| transition | ffmpeg | 转场、抽象概念、时间跨越 | 8-12秒 |
| content_scene | seedance | 具体历史场景、事件描述 | 10-15秒 |

### 工具选择原则

- **title_card**：始终使用 ffmpeg（免费，效果足够）
- **transition**：始终使用 ffmpeg（免费，Ken Burns 效果即可）
- **content_scene**：使用 seedance（需要 AI 生成动态内容）

## 镜头生成流程

### 步骤1：段落分析

**输入**：口播稿的段落列表（来自脚本解析）

**分析内容**：
1. 段落类型（开场、章节、内容、结尾）
2. 文本长度
3. 包含的时间信息
4. 包含的人物和地点

### 步骤2：镜头数量计算

**基于文本长度**：
- 中文阅读速度：约 3-4 字/秒
- 镜头时长：10-15 秒
- 每个镜头可承载：30-60 字

**计算公式**：
```
镜头数量 = ceil(文本长度 / 45)
```

**示例**：
- 文本长度 150 字 → 4 个镜头（每个约 37.5 字）
- 文本长度 200 字 → 5 个镜头（每个约 40 字）

### 步骤3：镜头类型分配

根据段落类型分配镜头：

| 段落类型 | 第一个镜头 | 中间镜头 | 最后一个镜头 |
|---------|-----------|---------|------------|
| 开场白 | title_card | transition | content_scene |
| 章节标题 | title_card | - | - |
| 过渡段落 | transition | transition | transition |
| 内容段落 | content_scene | content_scene | content_scene |
| 结尾语 | content_scene | transition | title_card |

### 步骤4：时长分配

**默认时长**：
- title_card: 8 秒
- transition: 10 秒
- content_scene: 12 秒

**动态调整**：
- 如果文本较长，增加镜头数量而非单个镜头时长
- 如果文本较短，减少镜头数量或使用较短时长
- 确保每个镜头时长在 5-15 秒之间

**分配算法**：
```
总时长 = 文本长度 / 4  （假设 4 字/秒）
每个镜头时长 = 总时长 / 镜头数量
限制：5秒 ≤ 每个镜头时长 ≤ 15秒
```

### 步骤5：时间码计算

**计算规则**：
1. 从 00:00 开始
2. 累加每个镜头的 duration
3. 格式：MM:SS-MM:SS

**计算示例**：
```
shot_001: duration=6  → time_code=00:00-00:06
shot_002: duration=8  → time_code=00:06-00:14
shot_003: duration=12 → time_code=00:14-00:26
...
```

### 步骤6：镜头编号

**编号规则**：
- 格式：shot_XXX（XXX 为 3 位数字，从 001 开始）
- 连续无断裂
- 按时间顺序排列

## 特殊场景处理

### 场景1：超长段落

**问题**：单个段落文本过长（> 500 字）

**解决方案**：
1. 自动拆分为多个子段落
2. 每个子段落独立生成镜头
3. 保持视觉连贯性（使用 reference_image）

### 场景2：无明确内容的段落

**问题**：段落内容为纯过渡或总结，没有具体场景

**解决方案**：
1. 使用 transition 类型的镜头
2. 生成抽象概念的视觉描述
3. 适当缩短时长（6-8 秒）

### 场景3：多个快速切换的场景

**问题**：段落包含多个独立场景（如列举多个事件）

**解决方案**：
1. 为每个独立场景生成一个镜头
2. 使用 transition 连接不同场景
3. 每个镜头时长控制在 8-10 秒

## 质量检查

生成镜头序列后，进行以下检查：

1. **时长检查**：所有镜头时长在 5-15 秒之间
2. **连续性检查**：时间码连续无断裂
3. **类型合理性**：镜头类型与内容匹配
4. **编号连续性**：镜头编号连续无断裂
5. **总时长合理性**：总时长与文本长度匹配（约 3-4 字/秒）
