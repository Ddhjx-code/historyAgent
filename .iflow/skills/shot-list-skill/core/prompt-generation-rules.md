# Prompt生成流程规范

## 概述

本文档定义了shot-list-skill中图像prompt生成的完整流程，将口播稿转换为高质量的叙事描述式prompt，确保历史准确性和视觉一致性。

## 核心原则

1. **叙事描述优先**：使用自然语言描述画面，而非关键词堆叠
2. **场景化思维**：构建完整的场景，包含环境、人物、动作、光线、氛围
3. **摄影术语融入**：自然融入摄影术语，而非生硬列出
4. **历史准确性**：基于特征库确保历史时代、人物、场景的准确性
5. **连贯性保障**：确保跨帧的人物、场景、光线一致性

## 总体流程

```
口播稿
  ↓
[阶段1：脚本解析] → 提取时代、人物、地点、事件、情绪
  ↓
[阶段2：节拍拆解] → 拆解为StoryBeat，选择9个锚点
  ↓
[阶段3：特征库查询] → 根据时代、身份、地点查询特征库
  ↓
[阶段4：Beat Board生成] → 为9个锚点生成视觉描述
  ↓
[阶段5：Sequence Board生成] → 将锚点扩展为连续动作
  ↓
[阶段6：一致性检查] → 检查人物、场景、光线连贯性
  ↓
[阶段7：Prompt优化] → 优化prompt质量和简洁性
  ↓
最终Prompt清单
```

## 详细流程

### 阶段1：脚本解析

参考：`core/script-parser-rules.md`

**输入**：口播稿文本

**输出**：
```json
{
  "script_title": "二二八事件",
  "era": "民国时期（1947）",
  "characters": ["林江迈", "专卖局查缉员"],
  "locations": ["天马茶房", "专卖局门口"],
  "events": ["查缉私烟", "冲突爆发"],
  "emotion_overall": "SOMBER",
  "metadata": {
    "date": "1947年2月27日",
    "location": "台北市太平町",
    "context": "二二八事件导火索"
  }
}
```

**关键步骤**：
1. 识别结构标记（开场白、第X部分、事件X）
2. 提取时代信息（通过历史关键词）
3. 识别人物身份（通过上下文）
4. 提取地点信息（通过地名或描述）
5. 分析整体情绪（通过情感关键词）

### 阶段2：节拍拆解

参考：`core/beat-breakdown-rules.md`

**输入**：脚本解析结果 + 口播稿全文

**输出**：StoryBeat列表（包含9个锚点）

**关键步骤**：
1. 按句子拆解为节拍
2. 识别节拍强度（HIGH/MEDIUM/LOW）
3. 分析观众收获（信息/情绪/悬念）
4. 选择9个关键锚点（覆盖叙事弧线）
5. 标注锚点类型（WORLDVIEW_ESTABLISH, INCITING_EVENT等）

**示例输出**：
```json
{
  "beats": [
    {
      "id": "beat_001",
      "description": "1947年2月27日傍晚，台北太平町天马茶房前，女烟贩林江迈正在摆摊",
      "audience_gains": ["时间地点", "人物身份"],
      "intensity": "LOW",
      "is_anchor": true,
      "anchor_type": "PROTAGONIST_STATUS",
      "emotion": "NEUTRAL"
    },
    {
      "id": "beat_002",
      "description": "专卖局查缉员突然出现，查缉林江迈的私烟",
      "audience_gains": ["冲突开始"],
      "intensity": "HIGH",
      "is_anchor": true,
      "anchor_type": "INCITING_EVENT",
      "emotion": "TENSE"
    }
  ]
}
```

### 阶段3：特征库查询

参考：`libraries/README.md`

**输入**：时代、人物身份、地点类型

**输出**：匹配的特征数据

**查询流程**：
1. **查询人物**：
   - 在`historical_characters.json`中查找`period`和`identity`匹配的条目
   - 提取`visual_features`、`clothing`、`pose`、`prompt_snippet`

2. **查询服饰**：
   - 在`historical_costumes.json`中查找`era`匹配的条目
   - 提取`prompt_snippet`、`materials`、`colors`

3. **查询建筑**：
   - 在`historical_architecture.json`中查找`era`匹配的条目
   - 提取`prompt_snippet`、`key_elements`、`atmosphere`

4. **查询地点**：
   - 在`historical_locations.json`中查找`era`和`name`匹配的条目
   - 提取`prompt_snippet`、`lighting`、`weather`、`time_of_day`

**示例输出**：
```json
{
  "character": {
    "name": "林江迈",
    "prompt_snippet": "一位40岁的台湾寡妇，身穿打补丁的粗布衣裳，脸上写满疲惫"
  },
  "costume": {
    "prompt_snippet": "民国时期平民服装，简单的粗布衣裳，破旧磨损"
  },
  "architecture": {
    "prompt_snippet": "台北老街，日本殖民时期木结构建筑，斜坡屋顶，日式装饰"
  },
  "location": {
    "prompt_snippet": "1947年台北市太平町天马茶房，街边烟纸摊，傍晚昏黄灯光"
  }
}
```

### 阶段4：Beat Board生成

参考：`core/beat-board-rules.md`

**输入**：锚点StoryBeat + 特征库数据

**输出**：Beat Board（9个格子的视觉描述）

**生成流程**：

1. **确定风格预设**：
   - 历史纪录片风格（默认）
   - 电影感风格（戏剧性强）
   - 纪实摄影风格（真实性高）

2. **选择景别**：
   - 根据锚点类型选择景别
   - WORLDVIEW_ESTABLISH → 远景/全景
   - PROTAGONIST_STATUS → 中景
   - INCITING_EVENT → 中近景
   - CLIMAX_CONFRONTATION → 近景/特写

3. **构建视觉描述**：
   - 使用标准结构（5个部分）
   - 融入特征库的`prompt_snippet`
   - 保持叙事描述式风格

**标准结构**：
```
[第1部分：风格、景别、环境]
[第2部分：主体、动作]
[第3部分：光线、光影]
[第4部分：色调、氛围]
[第5部分：画面参数]
```

**示例生成**：
```
历史纪录片风格，中景镜头，1947年台北太平町天马茶房前。
一位40岁的台湾寡妇，身穿打补丁的粗布衣裳，脸上写满疲惫，眼神中有对明天的担忧，蹲在路边摆摊，身体前倾，正在整理香烟摊位。
傍晚昏黄的街灯，透过薄雾洒在粗糙的路面上，形成斑驳的光影。
灰褐色和旧黄色的色调，破旧的木结构建筑和日式招牌，拥挤陈旧的生活气息，紧张混乱的氛围。
光圈 f/2.8，ISO 400，胶片质感。
```

### 阶段5：Sequence Board生成

参考：`core/sequence-board-rules.md`

**输入**：Beat Board + 锚点StoryBeat + 特征库数据

**输出**：Sequence Group（每个锚点扩展为4个格子的连续动作）

**生成流程**：

1. **确定继承关系**：
   - 设置`inheritance.reference_beat_image`引用对应的Beat Board图像
   - 确保人物/场景/光线一致

2. **设计结构**：
   - 默认使用"起—承—转—合"结构
   - 根据锚点类型调整结构

3. **扩展动作描述**：
   - 起：动作开始，引入场景
   - 承：动作发展，推进情节
   - 转：冲突或转折，情绪高潮
   - 合：动作结束，收束场景

4. **连贯性检查**：
   - 检查屏幕方向连续性
   - 检查动作连贯性
   - 检查轴线管理
   - 检查跳接风险

**示例生成**（INCITING_EVENT锚点）：
```
起：专卖局查缉员突然出现在街道尽头，制服深蓝色，表情严肃，快速向林江迈走来。傍晚街灯下，影子拉长，气氛骤然紧张。

承：查缉员走到林江迈面前，粗暴地翻检她的香烟摊位，林江迈惊慌失措，试图保护自己的货物。镜头逐渐推近，捕捉两人之间的紧张对峙。

转：查缉员推搡林江迈，她失去平衡倒在地上，额头撞到粗糙的路面，鲜血渗出。周围群众开始聚集，愤怒的声音逐渐高涨。

合：林江迈倒在地上痛苦呻吟，查缉员站在一旁冷漠观望，群众包围了他们，冲突即将升级。镜头拉远，展示整个混乱的现场。
```

### 阶段6：一致性检查

参考：`core/continuity-check-rules.md`（待创建）

**输入**：Beat Board + Sequence Board

**输出**：一致性检查报告

**检查清单**：

1. **人物一致性**：
   - [ ] 同一人物的年龄/性别/脸型一致
   - [ ] 同一人物的服饰类型/颜色一致
   - [ ] 同一人物的姿态合理连续

2. **场景一致性**：
   - [ ] 同一地点的建筑风格/材料一致
   - [ ] 同一场景的光线方向/时间一致
   - [ ] 同一场景的天气条件一致
   - [ ] 同一场景的氛围基调一致

3. **视觉连贯性**：
   - [ ] 屏幕方向连续（180度轴线规则）
   - [ ] 动作连贯（没有跳接）
   - [ ] 景别渐进（符合视觉节奏）

4. **历史准确性**：
   - [ ] 时代特征准确（服饰、建筑、道具）
   - [ ] 文化背景准确（礼仪、习俗）
   - [ ] 地理位置准确（建筑、景观）

**检查报告示例**：
```json
{
  "check_result": "PASS",
  "issues": [],
  "warnings": [
    "Shot 3-2的光线方向略有变化，建议调整"
  ],
  "suggestions": [
    "在Shot 3-1和3-2之间添加过渡镜头，避免跳接"
  ]
}
```

### 阶段7：Prompt优化

参考：`core/prompt-optimizer-rules.md`（待创建）

**输入**：Beat Board + Sequence Board + 一致性检查报告

**输出**：优化后的Prompt

**优化规则**：

1. **简洁性优化**：
   - 删除冗余描述
   - 合并相似元素
   - 保留核心视觉特征

2. **叙事流畅性优化**：
   - 确保描述逻辑连贯
   - 优化句子结构
   - 增强画面感

3. **技术参数优化**：
   - 确保摄影术语准确
   - 调整画面参数（光圈、ISO等）
   - 统一风格参数

4. **历史准确性优化**：
   - 修正历史错误
   - 补充历史细节
   - 确保文化尊重

**优化示例**：

## Prompt质量评估标准

### 评估维度

1. **历史准确性**（30分）：
   - 时代特征准确（10分）
   - 人物形象准确（10分）
   - 场景环境准确（10分）

2. **视觉描述质量**（30分）：
   - 场景化思维（10分）
   - 叙事流畅性（10分）
   - 画面感强度（10分）

3. **技术专业性**（20分）：
   - 摄影术语准确（10分）
   - 画面参数合理（10分）

4. **连贯性**（20分）：
   - 人物一致性（5分）
   - 场景一致性（5分）
   - 视觉连贯性（5分）
   - 跨帧协调性（5分）

### 评分标准

- 优秀（90-100分）：Prompt质量高，可直接使用
- 良好（80-89分）：Prompt质量较好，需微调
- 合格（70-79分）：Prompt基本可用，需修改
- 不合格（<70分）：Prompt不合格，需重新生成

## 常见问题和解决方案

### Q1：Prompt过长怎么办？

**解决方案**：
1. 删除冗余描述
2. 合并相似元素
3. 保留核心视觉特征（人物、动作、环境）
4. 删除次要细节（背景人物、远距离元素）

### Q2：历史特征库中缺少需要的特征怎么办？

**解决方案**：
1. 根据历史背景创建新条目
2. 使用通用特征描述
3. 查阅历史资料确保准确性
4. 咨询专业历史顾问

### Q3：如何处理跨时代的连续场景？

**解决方案**：
1. 明确标注每个场景的时代
2. 使用不同的视觉风格区分时代
3. 在过渡场景中明确时代变化
4. 确保每个时代的特征准确

### Q4：一致性检查发现错误怎么办？

**解决方案**：
1. 根据检查报告定位错误
2. 修改不一致的部分
3. 重新执行一致性检查
4. 确保所有检查项通过
