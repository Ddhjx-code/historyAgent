# Prompt生成规则（通用版）

## 概述

本文档定义了如何根据镜头类型和口播稿内容生成高质量的图像 prompt，确保 AI 能够理解并生成正确的画面。

## 核心原则

1. **具体化**：将抽象概念转化为具体的视觉描述
2. **完整性**：提供足够的信息让 AI 理解要画什么
3. **简洁性**：避免冗长，控制在合理长度内（< 200 字）
4. **通用性**：适用于各类口播稿，不依赖特定内容
5. **安全过滤**：过滤敏感词，避免生成失败

## 模型选择规则

**原则**：根据是否需要生成文字选择合适的模型

| needs_text | 模型 | 说明 |
|------------|------|------|
| true | doubao-seedream-4.5 | 高级模型，擅长文字生成，成本较高 |
| false | doubao-seedream-3.0 | 经济模型，成本较低，适用于纯画面 |

**步骤4.2：判断 needs_text**

判断镜头是否需要 AI 生成文字内容。

**判断逻辑**：

1. **title_card 类型**：`needs_text = true`
   - 标题卡总是需要生成文字

2. **其他类型**：检查 prompt 是否明确要求 AI 生成文字
   - 检查以下关键词：
     - `中央大字`：标题文字
     - `下方.*副标题`：副标题
     - `标题卡片`：标题卡
     - `文字.*显示`：文字显示
     - `字幕`：字幕
     - `屏幕上.*文字`：屏幕上的文字
   - 包含以上关键词 → `needs_text = true`
   - 否则 → `needs_text = false`

**注意事项**：

- ❌ **不要**将描述性的词当作需要生成文字：
  - "年份"、"数字"、"标记"：这些只是描述画面中的元素
  - "时间轴"、"地图"：这些是视觉元素，不是文字
- ✅ **只有**明确要求 AI 生成文字时才设置 `needs_text = true`

**示例**：

| Prompt | needs_text | 原因 |
|--------|-----------|------|
| `中央大字"历史上的今天"` | true | 明确要求生成标题文字 |
| `时间轴标记了公元138年、628年` | false | "标记"是描述画面元素，不是要求生成文字 |
| `古罗马帝国宫廷，哈德良皇帝` | false | 纯画面描述，无文字要求 |
| `下方副标题"2026年2月25日"` | true | 明确要求生成副标题 |

**判断函数（伪代码）**：
```python
def needs_text_generation(shot):
    # title_card 总是需要文字
    if shot['type'] == 'title_card':
        return True
    
    # 检查 prompt 中是否包含文字标记
    prompt = shot['tasks']['image']['parameters']['prompt']
    text_indicators = [r'标题', r'文字', r'字幕', r'年份', r'数字', r'\d+年', r'标记']
    
    for pattern in text_indicators:
        if re.search(pattern, prompt):
            return True
    
    return False
```

**成本优化**：
- 使用经济模型（3.0）可以降低约 30-40% 的图像生成成本
- 只有需要文字的镜头才使用高级模型（4.5）

## 敏感词过滤规则

### 敏感词列表

以下词汇会导致图像生成失败，必须过滤：

| 敏感词 | 替换为 | 说明 |
|--------|--------|------|
| 革命 | ** | 政治敏感词 |
| 台湾 | ** | 政治敏感词 |
| 共产党 | ** | 政治敏感词 |
| 苏维埃 | ** | 政治敏感词 |
| 共产主义 | ** | 政治敏感词 |

### 过滤规则

1. **替换策略**：将敏感词替换为 `**`
2. **应用范围**：所有 prompt 文本
3. **过滤时机**：在生成 prompt 后立即过滤

**过滤函数（伪代码）**：
```python
sensitive_words = ['革命', '台湾', '共产党', '苏维埃', '共产主义']

def filter_sensitive_text(text):
    filtered_text = text
    for word in sensitive_words:
        filtered_text = filtered_text.replace(word, '**')
    return filtered_text
```

### 示例

**原文**：
```
历史纪录片风格，章节标题卡片。中央大字"第三部分：工业革命与近现代的创新"。
```

**过滤后**：
```
历史纪录片风格，章节标题卡片。中央大字"第三部分：工业**与近现代的创新"。
```

**原文**：
```
历史纪录片风格，苏里南首都帕拉马里博，1980年。革命军士兵手持武器。
```

**过滤后**：
```
历史纪录片风格，苏里南首都帕拉马里博，1980年。**军士兵手持武器。
```

## 完整生成流程

### 阶段4：Prompt 生成与优化

```
生成 prompt
    ↓
判断 needs_text
    ├─ true → 使用 doubao-seedream-4.5
    └─ false → 使用 doubao-seedream-3.0
    ↓
过滤敏感词
    ↓
设置 needs_text 字段
    ↓
设置 model 字段
    ↓
最终 prompt
```

### 输出参数

每个镜头的 image 任务应包含：

```json
{
  "tool": "seedream",
  "model": "doubao-seedream-4.5 或 doubao-seedream-3.0",
  "parameters": {
    "prompt": "过滤后的 prompt",
    "size": "1920x1080",
    "n": 1,
    "reference_image": "string 或 null",
    "needs_text": true 或 false,
    "seed": "integer 或 null"
  }
}
```

## 通用 Prompt 结构

```
[视觉风格]，[核心描述]。
[具体细节]，[环境氛围]。
电影质感，16:9横屏，[整体基调]。
```

### 结构说明

| 部分 | 说明 | 示例 |
|------|------|------|
| [视觉风格] | 整体美术风格 | 历史纪录片风格 |
| [核心描述] | 画面主体和核心内容 | 古罗马帝国宫殿，哈德良皇帝坐在宝座上 |

| [整体基调] | 整体感觉 | 庄严肃穆 |

## 各类型 Prompt 生成规则

### 1. 标题卡 Prompt

**用途**：节目标题、章节标题、结尾感谢

**模板**：
```
[视觉风格]，标题卡片设计。
中央大字"[标题内容]"，[副标题或装饰文字]。
[装饰元素描述]，[背景描述]。
电影质感，16:9横屏，[整体氛围]。
```

**生成规则**：
1. 提取段落标题作为主标题
2. 提取日期或副标题作为副标题
3. 根据段落类型选择装饰元素：
   - 开场/结尾：时光隧道、星光效果
   - 章节：金色边框、历史纹样
4. 背景使用抽象或装饰性元素

**示例**：
```
历史纪录片风格，标题卡片设计。
中央大字"历史上的今天"，下方副标题"2026年2月25日"。
金色边框装饰，背景是历史时光隧道效果，展现从古代到现代的时间流动。
电影质感，16:9横屏，庄重大气。
```

---

### 2. 转场镜头 Prompt

**用途**：过渡、抽象概念、时间跨越

**核心原则**：将抽象概念转化为具体的视觉元素

**模板**：
```
[视觉风格]，[抽象概念的具体化描述]。
[具体视觉元素列举]，[视觉连接方式]。
电影质感，16:9横屏，[整体氛围]。
```

**生成规则**：
1. 识别抽象概念（如"历史长河"、"时间跨越"）
2. 转化为具体视觉元素：
   - 时间轴 → 带有年份标记的时间线
   - 历史长河 → 河流、时间隧道、星光
   - 文明交融 → 不同文化的元素交织
3. 列举关键的时间点或事件（从口播稿提取）
4. 使用连接词建立视觉联系（金色光芒、流动的线条）

**关键技巧**：
- **不要说**："时间轴延伸"、"金色光点标记"
- **要说**："时间轴从左到右延伸，标记了公元138年、628年、645年等年份。每个年份用金色光点标记"

**示例**：
```
历史纪录片风格，横向时间轴画面。
背景是深邃的宇宙星空。时间轴从左到右延伸，标记了公元138年（罗马帝国）、628年（波斯萨珊）、645年（玄奘取经）、1836年（左轮手枪）、1901年（美国钢铁）、1921年（苏联扩张）、1961年（科威特独立）、1980年（苏里南革命）、2026年（中国外交部）。每个年份用金色光点标记，金色光芒连接各个光点，展现历史进程。
电影质感，16:9横屏，宏大史诗感。
```

### 3. 内容场景 Prompt

**用途**：具体的历史场景、事件描述

**模板**：
```
[视觉风格]，[时代描述]。
[人物描述]，[人物动作]，[场景环境]。
[光影氛围]，[色调描述]。
电影质感，16:9横屏，[整体氛围]。
```

**生成规则**：
1. **时代描述**：从口播稿提取时间信息（如"古罗马帝国时代"、"唐代"）
2. **人物描述**：从口播稿提取人物信息（姓名、身份、服饰）
3. **人物动作**：从口播稿提取人物的具体动作
4. **场景环境**：从口播稿提取地点信息和环境描述
5. **光影氛围**：根据场景类型推断（室内/室外、白天/夜晚）
6. **色调描述**：根据时代和场景推断（暖色/冷色、鲜艳/素雅）

**信息提取方法**：
- 从脚本解析结果中获取时间、人物、地点信息
- 从文本内容中提取动作和环境描述
- 使用原文的关键词，避免添加未经证实的细节

**示例**：
```
历史纪录片风格，古罗马帝国时代。
哈德良皇帝身着紫色镶边托加长袍，头戴橄榄枝冠冕，坐在大理石宝座上。宫廷内罗马柱林立，马赛克地板，墙上挂着罗马鹰旗。温暖的地中海阳光从高窗射入，形成戏剧性光影。
电影质感，16:9横屏，庄严肃穆。
```

**示例2（无明确人物）**：
```
历史纪录片风格，19世纪美国工业时代。
巨大的钢铁工厂内部，高耸的烟囱冒着浓烟，钢铁熔炉发出橙红色的光芒。工人们正在操作机器，火花四溅。蒸汽机的轰鸣声仿佛能透过画面感受到。
电影质感，16:9横屏，工业化氛围。
```

## 信息提取与填充

### 从口播稿提取信息

使用脚本解析阶段提取的信息：

```python
# 伪代码示例
def generate_content_scene_prompt(shot, script_info):
    # 提取时间信息
    time_info = extract_time_info(shot.content, script_info.time_info)
    era_description = f"{time_info.era}时代" if time_info.era else "历史时期"

    # 提取人物信息
    characters = extract_characters(shot.content, script_info.characters)
    if characters:
        character_desc = f"{characters[0].name}，{characters[0].identity}"
        action_desc = extract_action(shot.content, characters[0])
    else:
        character_desc = ""
        action_desc = extract_action(shot.content, None)

    # 提取地点信息
    locations = extract_locations(shot.content, script_info.locations)
    if locations:
        location_desc = locations[0].description
    else:
        location_desc = ""

    # 构建 prompt
    prompt = f"历史纪录片风格，{era_description}。"
    if character_desc:
        prompt += f"{character_desc}，{action_desc}。"
    if location_desc:
        prompt += f"{location_desc}。"
    prompt += "电影质感，16:9横屏。"

    return prompt
```

## 常见问题与解决

### 问题1：抽象概念无法可视化

**错误示例**：
```
历史纪录片风格，历史的见证。
时间轴延伸，见证文明进步。
电影质感，16:9横屏。
```

**问题**："历史长河"、"文明进步"无法直接生成

**正确做法**：
```
历史纪录片风格，横向时间轴画面。
时间轴标记了从古代到现代的关键事件，金色光芒连接各个时代。
电影质感，16:9横屏，宏大史诗感。
```

### 问题2：信息过少，画面模糊

**错误示例**：
```
历史纪录片风格，罗马皇帝。
坐在宝座上。
电影质感，16:9横屏。
```

**问题**：没有具体的视觉细节

**正确做法**：
```
历史纪录片风格，古罗马帝国时代。
哈德良皇帝身着紫色镶边托加长袍，头戴橄榄枝冠冕，坐在大理石宝座上。宫廷内罗马柱林立，马赛克地板。
电影质感，16:9横屏，庄严肃穆。
```

### 问题3：信息过多，prompt 冗长

**错误示例**：
```
历史纪录片风格，古罗马帝国时代，公元138年2月25日。哈德良皇帝，罗马帝国的第五位皇帝，也是"五贤帝"中的第三位，他身着紫色镶边托加长袍，头戴橄榄枝冠冕，坐在大理石宝座上。宫廷内罗马柱林立，马赛克地板描绘着神话故事，墙上挂着罗马鹰旗，高窗射入温暖的阳光，形成戏剧性光影。电影质感，16:9横屏，庄严肃穆。
```

**问题**：包含了过多的历史背景信息

**正确做法**：
```
历史纪录片风格，古罗马帝国时代。
哈德良皇帝身着紫色镶边托加长袍，头戴橄榄枝冠冕，坐在大理石宝座上。宫廷内罗马柱林立，马赛克地板，墙上挂着罗马鹰旗。温暖的地中海阳光从高窗射入，形成戏剧性光影。
电影质感，16:9横屏，庄严肃穆。
```

## Prompt 质量检查

生成 prompt 后，进行以下检查：

1. **具体性检查**：是否包含具体的视觉元素（人物、服饰、场景、光线）
2. **完整性检查**：是否提供足够的信息让 AI 理解要画什么
3. **简洁性检查**：长度是否控制在合理范围内（< 200 字）
4. **通用性检查**：是否适用于各类口播稿，不依赖特定内容
