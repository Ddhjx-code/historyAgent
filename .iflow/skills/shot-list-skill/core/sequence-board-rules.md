# Sequence Board生成规则

本文档定义大模型生成Sequence Board（四宫格/可变格）时应遵循的规则。

## Sequence Board目的

Sequence Board（四宫格/可变格展开板）用于：
1. **展开段落内部**：围绕Beat Board格子展开连续动作
2. **保持一致性**：继承Beat Board的人物/场景/光色
3. **确保连贯性**：检查屏幕方向、轴线、跳接规则

## 结构规则

### 默认结构（四格）

| 格子 | 类型 | 功能 |
|------|------|------|
| 1 | 起 | 建立局部空间与动机/威胁 |
| 2 | 承 | 行动推进（动作开始） |
| 3 | 转 | 冲突/受阻/反转（代价出现） |
| 4 | 合 | 结果落点 + 下一步钩子 |

### 动作戏结构（可替换）

| 格子 | 类型 | 功能 |
|------|------|------|
| 1 | 预备 | 准备动作 |
| 2 | 出手 | 动作开始 |
| 3 | 命中/受击 | 撞击瞬间 |
| 4 | 反应/余势 | 结果反应 |

### 格数选择

| 复杂度 | 格数 | 说明 |
|--------|------|------|
| 简单信息 | 2格 | 信息交代、场景转换 |
| 常规段落 | 4格 | 默认结构 |
| 动作复杂 | 6格 | 必须保证动作连续与空间可读 |

## 继承性规则

### 必须继承的元素

```yaml
character: "人物ID"           # 人物ID
clothing: "服装描述"          # 服装
location: "地点"              # 地点
lighting: "光照类型"          # 光照
atmosphere: "氛围"            # 氛围
```

### 变化说明

如果发生必须变化，显式说明原因：

```yaml
changes:
  - type: "injury"
    description: "林江迈头部受伤，额头有血迹"
    reason: "被枪托击打"
  - type: "lighting"
    description: "光线变暗"
    reason: "进入室内"
```

## 镜头生成规则

### 镜头序列

| 镜头 | 类型 | 景别 | 动作阶段 | 光线变化 |
|------|------|------|---------|---------|
| 1 | 起 | 中景 | start | 柔光，光线均匀 |
| 2 | 承 | 中近景 | middle | 侧光，逐渐增强 |
| 3 | 转 | 特写 | climax | 硬光，强烈对比 |
| 4 | 合 | 中景 | aftermath | 漫射光，柔和 |

### 提示词生成

**参考**：`templates/prompt-templates.md`

### 镜头1（起）- 使用模板2

```
[场景环境描述]，[天气条件]，[时间]。[人物名称] [出现在场景的方式]，[外貌特征]，[服饰描述]。[初始情绪]，[氛围描述]。[光线细节]，[背景元素]，[视觉焦点]。
```

### 镜头2（承）- 使用模板3

```
[主体A] [动作描述]，[主体B] [反应描述]。[互动细节]，[动作连续性]。[情绪升级/变化]，[氛围转变]。[镜头变化]，[视觉重点]，[背景变化]。
```

### 镜头3（转）- 使用模板4

```
[冲突发生]，[关键动作]。[强烈动作描述]，[激烈互动]。[情绪爆发]，[极度紧张/悲伤/愤怒]。[特写细节]，[视觉冲击]，[戏剧性光影]。
```

### 镜头4（合）- 使用模板5

```
[动作结果]，[最终状态]。[人物状态]，[场景变化]。[情绪沉淀]，[余韵]。[镜头拉远]，[全貌展示]，[场景收束]。
```

## 连贯性规则

**参考**：`core/continuity-check-rules.md`

### 屏幕方向一致

**规则**：面向、运动方向、进出画方向应保持稳定

**检查项**：
- [ ] 角色在前一镜头朝左移动，下一镜头不能突然反向朝右
- [ ] 角色从画面右侧退出，下一镜头应从画面左侧入镜
- [ ] 遵守180度轴线规则

### 动作中段切

**规则**：优先在动作明显运动中段/高峰切镜头

**检查项**：
- [ ] 避免在动作未起未落的尴尬瞬间切换
- [ ] 动作中段切让观众大脑自然将两个镜头的动作连接起来
- [ ] 确保动作阶段连续（start→middle→climax→aftermath）

**示例**：
- ✅ 挥拳动作在拳头到达最高点时切换
- ❌ 在拳头收回后切换

### 轴线管理

**规则**：对话/对峙先建立轴线与左右关系，机位尽量保持轴线同侧

**检查项**：
- [ ] 已建立轴线：机位保持同侧
- [ ] 必须越轴时，使用过渡手段（角色移动、中性过渡镜头）
- [ ] 轴线变化有明确原因

### 避免跳接

**规则**：相邻镜头如果变化太小，容易产生跳接抖动感

**检查项**：
- [ ] 30度法则：切换镜头时机位至少偏移30度或推远/拉近适当距离
- [ ] 景别变化：远↔近，避免相同景别直接切换
- [ ] 景别渐进性：符合视觉节奏

## 连贯性检查报告

**参考**：`core/continuity-check-rules.md`

### 检查清单

**人物一致性**：
- [ ] 同一人物的年龄/性别/脸型一致
- [ ] 同一人物的服饰类型/颜色一致
- [ ] 同一人物的姿态合理连续
- [ ] 同一人物的表情符合情绪发展

**场景一致性**：
- [ ] 同一地点的建筑风格/材料一致
- [ ] 同一场景的光线方向/时间一致
- [ ] 同一场景的天气条件一致
- [ ] 同一场景的氛围基调一致
- [ ] 同一场景的色调一致

**视觉连贯性**：
- [ ] 屏幕方向连续（180度轴线规则）
- [ ] 动作连贯（没有跳接）
- [ ] 景别渐进（符合视觉节奏）
- [ ] 构图变化合理
- [ ] 镜头运动合理

**历史准确性**：
- [ ] 时代特征准确（服饰、建筑、道具）
- [ ] 文化背景准确（礼仪、习俗、称呼）
- [ ] 地理位置准确（建筑、景观、气候）

### 检查项

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 屏幕方向 | 面向、运动方向一致 | passed/warning/failed |
| 动作连贯性 | 动作阶段连续（start→middle→climax→aftermath） | passed/warning/failed |
| 轴线管理 | 轴线稳定或使用过渡 | passed/warning/failed |
| 跳接风险 | 角度/景别变化足够 | passed/warning/failed |
| 人物一致性 | 人物外观/服装/姿态一致 | passed/warning/failed |
| 场景一致性 | 场景光线/色调/氛围一致 | passed/warning/failed |

### 检查报告格式

```json
{
  "check_scope": "sequence_group_01",
  "overall_result": "PASS",
  "score": 92,
  "details": {
    "character_consistency": {"result": "PASS", "score": 95},
    "scene_consistency": {"result": "PASS", "score": 90},
    "visual_continuity": {"result": "PASS", "score": 90},
    "historical_accuracy": {"result": "PASS", "score": 95}
  },
  "issues": [],
  "warnings": [
    "Shot 3-2的光线方向略有变化"
  ],
  "suggestions": [
    "在Shot 3-1和3-2之间添加过渡镜头，避免跳接"
  ]
}
```

## 验收标准

生成Sequence Board后必须检查：

**动作与因果**：
- [ ] 起手—推进—冲突—落点是否清楚？
- [ ] 动作是否连续、空间是否可读？
- [ ] 每个镜头的目的是否明确（信息/情绪/悬念/转折）？

**一致性**：
- [ ] 是否继承Beat Board对应格的人物/场景/光色？
- [ ] 变化是否有明确原因说明？
- [ ] 人物外观/服装/姿态是否一致？
- [ ] 场景光线/色调/氛围是否一致？

**连贯性**：
- [ ] 屏幕方向/视线/轴线是否稳定？
- [ ] 越轴是否有过渡手段？
- [ ] 动作阶段是否连续（start→middle→climax→aftermath）？
- [ ] 是否使用动作中段切来提高衔接自然度？

**剪接风险**：
- [ ] 是否存在跳接风险（角度变化不足、景别变化不足）？
- [ ] 30度法则是否遵守？
- [ ] 景别变化是否合理？

**节奏与张力**：
- [ ] 段落内部是否有张弛与呼吸？
- [ ] 是否有信息控制（藏与露）与关键细节强调？
- [ ] 张力是否逐渐升级？

**承接性**：
- [ ] 最后一格是否能自然接到下一锚点/下一场景？
- [ ] 是否有明确的过渡或钩子？

**历史准确性**：
- [ ] 时代特征是否准确？
- [ ] 文化背景是否准确？
- [ ] 地理位置是否准确？

**质量评分**：
- [ ] 总体评分 ≥ 90分（优秀）
- [ ] 无严重问题（issues）
- [ ] 警告数量 ≤ 2个（warnings）

## 注意事项

1. **继承一致性**：必须继承Beat Board的人物/场景/光色
2. **动作连贯**：确保动作阶段连续（start→middle→climax→aftermath）
3. **避免跳接**：相邻镜头的角度/景别变化要足够
4. **节奏控制**：高强度段落后安排呼吸点（静镜/空镜）
5. **模板遵循**：严格按照模板2-5的结构生成Prompt
6. **特征库使用**：优先使用特征库的`prompt_snippet`，确保历史准确性
7. **简洁性**：每个镜头的Prompt长度控制在200字以内
8. **画面要素**：每个镜头的景别、主体、环境、光线、色调清晰
9. **叙事目的**：每个镜头的目的明确（信息/情绪/悬念/转折）
