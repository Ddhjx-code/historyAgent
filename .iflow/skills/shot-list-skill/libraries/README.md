# 历史特征库使用指南

## 概述

历史特征库提供结构化的历史知识，确保分镜和图像prompt的历史准确性和人物/场景一致性。

## 文件结构

```
libraries/
├── character-library.md           # 人物特征库设计文档
├── historical_characters.json    # 人物数据（10个人物）
├── historical_costumes.json      # 服饰数据（10类服饰）
├── historical_architecture.json  # 建筑数据（7类建筑）
└── historical_locations.json     # 地点数据（10个地点）
```

## 查询规则

### 1. 按时代查询

从口播稿解析出时代信息（`era`），然后在对应的JSON文件中查找匹配的人物、服饰、建筑、地点。

**示例**：
- 输入时代："民国时期（1947）"
- 匹配人物：林江迈、陈文溪、专卖局查缉员
- 匹配服饰：民国平民服装、政府制服
- 匹配建筑：台北老街
- 匹配地点：天马茶房、专卖局门口

### 2. 按身份/职业查询人物

从口播稿中识别人物身份，在`historical_characters.json`中查找`identity`字段。

**示例**：
- 身份："烟纸摊贩" → 林江迈
- 身份："皇帝" → 哈德良皇帝、唐太宗、明成祖朱棣

### 3. 按地点类型查询

从口播稿中识别地点类型，在`historical_locations.json`中查找`type`字段。

**示例**：
- 地点类型："商业场所" → 天马茶房
- 地点类型："皇家宫殿" → 罗马帝国宫殿、紫禁城

### 4. 提取prompt_snippet

每个特征条目都包含`prompt_snippet`字段（≤50字），直接用于图像prompt。

**示例**：
- 林江迈的prompt_snippet：
  ```
  一位40岁的台湾寡妇，身穿打补丁的粗布衣裳，脸上写满疲惫，眼神中有对明天的担忧
  ```

## 一致性检查要点

### 人物一致性

1. **视觉特征一致性**：
   - 同一人物的年龄、性别、脸型、表情必须一致
   - 使用`visual_features`字段定义的标准

2. **服饰一致性**：
   - 同一人物的服饰类型、材质、颜色必须一致
   - 使用`clothing`字段定义的标准

3. **姿态一致性**：
   - 参考角色的默认姿态，必要时可使用备选姿态
   - 避免同一场景中人物姿态冲突

### 场景一致性

1. **建筑一致性**：
   - 同一地点的建筑风格、材料、颜色必须一致
   - 使用`architecture`和`location`特征库确保准确

2. **光线一致性**：
   - 同一场景的光源方向、时间、天气必须一致
   - 使用`locations.json`中的`lighting`、`weather`、`time_of_day`

3. **氛围一致性**：
   - 同一场景的情绪基调、氛围描述必须一致
   - 使用`locations.json`中的`atmosphere`

### 跨帧一致性检查清单

- [ ] 人物年龄/性别/脸型一致
- [ ] 人物服饰类型/颜色一致

- [ ] 无跳接风险

## Prompt生成流程

### Beat Board Prompt生成

1. **识别锚点类型**：
   - 使用`AnchorType`枚举确定锚点类型
   - 参考`beat-breakdown-rules.md`的锚点定义

2. **查询特征库**：
   - 根据时代、身份、地点类型查询特征库
   - 提取匹配的人物、服饰、建筑、地点

3. **构建Prompt**：
   - 使用`beat-board-rules.md`的标准结构
   - 融入特征库的`prompt_snippet`
   - 保持叙事描述式风格

### Sequence Board Prompt生成

1. **继承Beat Board**：
   - 使用`inheritance`字段引用对应的Beat Board图像
   - 保持人物/场景/光线一致

2. **扩展动作描述**：
   - 基于"起—承—转—合"结构扩展
   - 参考`sequence-board-rules.md`的结构定义

3. **连贯性检查**：
   - 执行跨帧一致性检查清单
   - 确保动作连续、方向正确

## 特征库扩展规则

当需要添加新的人物、服饰、建筑、地点时：

1. **创建新条目**：
   - 使用标准数据结构（参考`character-library.md`）
   - 确保所有必填字段完整
   - 编写准确的`prompt_snippet`（≤50字）

2. **验证准确性**：
   - 查阅历史资料确保准确性
   - 参考历史时期的视觉资料
   - 使用专业历史顾问审核

3. **测试一致性**：
   - 与现有特征库进行交叉验证
   - 确保时代、风格、文化背景一致
   - 测试prompt生成效果

## 注意事项

1. **prompt_snippet长度限制**：确保每个`prompt_snippet`不超过50字，避免prompt过长
2. **时代准确性**：严格区分不同时代的特征，避免时代错乱
3. **文化尊重**：准确描述不同文化背景的特征，避免刻板印象
4. **视觉可用性**：确保特征描述能够转化为视觉元素
5. **简洁性**：优先使用简洁明确的描述，避免冗长

## 常见问题

### Q1：如何处理跨时代的人物？

A：使用`period`字段明确时代，同一人物在不同时代可能需要创建多个条目。

### Q2：如何处理模糊的身份描述？

A：优先匹配精确的身份描述，如无法匹配，使用相近的身份类型。

### Q3：如何处理特征库中缺少的特征？

A：根据历史背景创建新条目，或使用通用的特征描述，确保历史准确性。

### Q4：如何处理prompt_snippet过长的问题？

A：优先保留关键视觉特征（年龄、性别、服饰、表情），删除次要描述。
