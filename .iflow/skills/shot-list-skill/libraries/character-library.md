# 历史人物特征库

## 概述

历史人物特征库是shot-list-skill的核心资源，用于确保历史视频中的人物描述准确、一致、丰富。

## 目标

1. **准确性**：确保历史人物的外貌、服装、姿态描述符合历史事实
2. **一致性**：同一人物在不同镜头中保持外观一致
3. **丰富性**：提供详细的视觉特征描述，便于AI生成高质量图像
4. **可扩展性**：支持灵活添加新人物，覆盖更多历史时代

## 字段说明

### 基本信息

| 字段 | 说明 | 示例 |
|------|------|------|
| character_id | 唯一标识符 | `lin_jiangmai_1947` |
| name | 人物姓名 | `林江迈` |
| period | 历史时期 | `民国时期（1947）` |
| identity | 身份/职业 | `烟纸摊贩` |

### 视觉特征（visual_features）

| 字段 | 说明 | 示例 |
|------|------|------|
| age | 年龄描述 | `40岁左右` |
| gender | 性别 | `女性` |
| face_shape | 脸型描述 | `饱经风霜的脸庞，颧骨突出` |
| expression | 表情描述 | `疲惫、担忧、充满生存压力` |
| body_type | 体型描述 | `瘦弱、营养不良` |
| hair | 发型描述 | `黑色头发，简单的发髻` |
| skin_tone | 肤色描述 | `黄褐色，长期户外工作晒黑` |

### 服装（thing）

| 字段 | 说明 | 示例 |
|------|------|------|
| type | 服装类型 | `打补丁的粗布衣裳` |
| material | 材质 | `粗棉布` |
| color | 颜色 | `灰褐色` |
| condition | 新旧状态 | `破旧、磨损、多处补丁` |
| accessories | 配饰列表 | `无` |

### 姿态（pose）

| 字段 | 说明 | 示例 |
|------|------|------|
| default | 默认姿态 | `蹲在路边摆摊，身体前倾` |
| alternatives | 备选姿态 | `蹲坐防御姿态`, `站立疲惫姿态` |

### Prompt片段（prompt_snippet）

**用途**：可直接插入到完整的图像提示词中

**格式**：简短、具体、视觉化

**示例**：
```
一位40岁的台湾寡妇，身穿打补丁的粗布衣裳，脸上写满疲惫，眼神中有对明天的担忧
```

### 历史背景（historical_context）

**用途**：帮助理解人物在历史中的地位和作用

**内容**：简要说明人物的历史背景、重要事件、影响

**示例**：
```
林江迈是1947年台湾"二二八事件"的导火索人物。她是一位40岁的寡妇，
为了养活两个孩子在台北市太平町摆烟纸摊。2月27日，专卖局查缉员
殴打她的事件引发了大规模民众反抗，最终演变为影响台湾社会的重大事件。
```

## 使用方法

### 1. 在Prompt中引用人物

```markdown
# 步骤1：识别人物
从口播稿中提取人物信息：林江迈、40岁、寡妇、烟纸摊贩

# 步骤2：查询特征库
在historical_characters.json中查找匹配的人物

# 步骤3：提取特征
提取林江迈的视觉特征、服装、姿态描述

# 步骤4：生成Prompt
历史纪录片风格。中景，1947年台北市太平町的黄昏街头。
[人物描述：使用prompt_snippet]。
[场景描述]，[动作描述]。
电影质感，16:9横屏。
```

### 2. 保持一致性

```markdown
# 同一人物在不同镜头中：
- 使用相同的character_id
- 使用相同的visual_features
- 使用相同的clothing描述
- 只允许在必要时变化pose

# 示例：
镜头1（林江迈摆摊）：
- pose: "蹲在路边摆摊"
- prompt_snippet: "一位40岁的台湾寡妇，身穿打补丁的粗布衣裳..."

镜头2（林江迈被殴打）：
- pose: "蹲坐防御姿态"
- prompt_snippet: "一位40岁的台湾寡妇，身穿打补丁的粗布衣裳..."
```

### 3. 扩展特征库

当遇到新的历史人物时：

1. **验证历史准确性**：查阅历史资料，确保描述准确
2. **填写所有字段**：包括基本信息、视觉特征、服装、姿态等
3. **编写Prompt片段**：确保简短、具体、视觉化
4. **添加历史背景**：帮助理解人物在历史中的地位
5. **更新版本号**：记录特征库的更新

## 质量标准

### 人物信息完整性

- [ ] 所有必需字段都已填写
- [ ] visual_features至少包含5个维度
- [ ] thing包含type、material、color
- [ ] pose包含default和至少1个alternative
- [ ] prompt_snippet简短、具体、视觉化（≤50字）
- [ ] historical_context准确、简洁

### 历史准确性

- [ ] 人物基本信息符合历史记载
- [ ] 服装符合时代特征
- [ ] 姿态符合身份和场景
- [ ] 没有明显的历史错误

### 可用性

- [ ] Prompt片段可以直接插入到完整的Prompt中
- [ ] 人物描述具体、清晰，便于AI理解
- [ ] 同一人物的描述在前后镜头中保持一致

### 版本管理

### 新增人物流程

1. 提议新增人物（基于口播稿需求）
2. 查阅历史资料，收集信息
3. 填写人物数据结构
4. 审核历史准确性
5. 更新版本号和changelog
6. 提交审核

## 注意事项

1. **避免过度细化**：不要为每个细微差别创建单独的人物条目
2. **保持简洁**：Prompt片段应该简短、具体，避免过长
3. **历史优先**：始终以历史记载为准，避免主观臆造
4. **时代一致性**：确保人物的服装、姿态符合时代特征
5. **定期更新**：根据口播稿需求不断扩展特征库覆盖范围
