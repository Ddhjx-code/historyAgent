## 连贯性规则检查

### 规则说明

参考方法论补充.md的[连贯性与剪接规则库]：

1. **屏幕方向一致**: 面向、运动方向、进出画方向应保持稳定
2. **动作中段切**: 优先在动作明显运动中段/高峰切镜头
3. **轴线管理**: 尽量保持轴线同侧，越轴时使用过渡手段
4. **避免跳接**: 角度/景别变化不足时避免直接切换
5. **视觉中心延续**: 保持主体在画面位置的延续

## 提示词生成规则

### 继承性规则

每个镜头必须继承Beat Board对应格的关键描述：

1. **人物外观**: 使用相同的character_id和视觉特征
2. **服装状态**: 使用相同的服装描述
3. **场景环境**: 使用相同的地点和建筑风格
4. **光色基调**: 使用相同的光照类型和色调

### 结构规则

**默认结构（四格）**:
- **起**: 建立局部空间与动机/威胁
- **承**: 行动推进（动作开始）
- **转**: 冲突/受阻/反转（代价出现）
- **合**: 结果落点 + 下一步钩子

**动作戏结构（可替换）**:
- **预备**: 准备动作
- **出手**: 动作开始
- **命中/受击**: 撞击瞬间
- **反应/余势**: 结果反应

**格数可变（按复杂度）**:
- 简单信息：2格足够
- 常规段落：4格
- 动作复杂：6格（必须保证动作连续与空间可读）

### 变化说明

如果发生必须变化，显式说明原因：

```yaml
changes:
  - type: "injury"
    description: "林江迈头部受伤，额头有血迹"
    reason: "被枪托击打"
  - type: "lighting"
    description: "光线变暗"
    reason: "进入室内"
```

### 张力营造

**构图营造张力**:
- 空间压迫：前景挤压、阴影压迫
- 空间释放：留白、开阔构图
- 对比制造张力：明暗对比、色调冷暖对比

**节奏营造张力**:
- 快节奏：镜头时长短、切换频率高
- 慢节奏：镜头时长长、停留时间长
- 呼吸点：高强度段落后安排静镜

## 验收清单

生成Sequence Board后必须检查：

- [ ] **动作与因果**: 起手—推进—冲突—落点是否清楚？
- [ ] **动作与因果**: 动作是否连续、空间是否可读？
- [ ] **一致性**: 是否继承Beat Board对应格的人物/场景/光色？
- [ ] **一致性**: 变化是否有明确原因说明？
- [ ] **连贯性**: 屏幕方向/视线/轴线是否稳定？
- [ ] **连贯性**: 越轴是否有过渡手段？
- [ ] **剪接风险**: 是否存在跳接风险（角度变化不足、景别变化不足）？
- [ ] **剪接风险**: 是否使用动作中段切来提高衔接自然度？
- [ ] **节奏与张力**: 段落内部是否有张弛与呼吸？
- [ ] **节奏与张力**: 是否有信息控制（藏与露）与关键细节强调？
- [ ] **承接性**: 最后一格是否能自然接到下一锚点/下一场景？
